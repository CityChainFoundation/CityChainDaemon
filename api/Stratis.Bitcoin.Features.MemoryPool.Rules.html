Namespace Stratis.Bitcoin.Features.MemoryPool.Rules

Classes

CheckAllInputsMempoolRule

Validate inputs against previous transactions. Checks against Standard and P2SH

CheckAncestorsMempoolRule

Validates the ancestors of a memory pool entry. Checks that the number of ancestors isn't too large. Checks for a transaction that spends outputs that would be replaced by it.

CheckCoinViewMempoolRule

Validates the transaction with the coin view. Checks if already in coin view, and missing and unavailable inputs.

CheckConflictsMempoolRule

Check for conflicts with in-memory transactions. If a conflict is found it is added to the validation context.

CheckFeeMempoolRule

Validates the transaction fee is valid. Checks whether the fee meets minimum fee, free transactions have sufficient priority, and absurdly high fees.

CheckRateLimitMempoolRule

Validates the rate limit. Currently not implemented.

CheckReplacementMempoolRule

Check if transaction can replace others. Only transactions that increase fees over previous transactions are accepted. There is a restriction on the maximum number of transactions that would be replaced. The new transaction must have all inputs confirmed. The new transaction must have sufficient fees to pay for it's bandwidth.

CheckSigOpsMempoolRule

Check that the transaction doesn't have an excessive number of sigops.

CheckTxOutDustRule

This rule ensures that the transaction does not contain any outputs which are considered dust.

CreateMempoolEntryMempoolRule

Creates a memory pool entry in the validation context. Validates the transactions can be mined, and the pay to script hashs are standard. Calculates the fees related to the transaction.
