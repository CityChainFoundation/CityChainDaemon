Class ConcurrentFixedSizeQueue<T>

Non locking Concurrent Fixed Size Queue. This implementation is a lose fixed size queue, because it may sometime exceed the number of items because it wraps a ConcurrentQueue and that is a lock-free concurrent queue implementation, so even if there is a chance it may exceed maxSize, it serves the purpose of circular buffer to hold a limited set of updated elements.
