GetHeadersPayload getHeadersPayload

Payload of "getheaders" message to process.
Returns
Type Description
Task
Remarks

"getheaders" message is sent by the peer in response to "headers" message until an empty array is received.

This payload notifies peers of our current best validated height, which is held by consensus tip.

If the peer is behind/equal to our best height an empty array is sent back.

ProcessHeadersAsync(INetworkPeer, List<BlockHeader>)

Processes "headers" message received from the peer.

Declaration

Parameters
Type Name Description
INetworkPeer peer

Peer from which the message was received.
List<BlockHeader> headers

List of headers to process.
Returns
Type Description
Task
Remarks

"headers" message is sent in response to "getheaders" message or it is solicited by the peer when a new block is validated (unless in IBD).

When we receive "headers" message from the peer, we can adjust our knowledge of the peer's view of the chain. We update its pending tip, which represents the tip of the best chain we think the peer has.

ProcessInvAsync(INetworkPeer, List<InventoryVector>)

Processes "inv" message received from the peer.

Declaration
protected Task ProcessInvAsync(INetworkPeer peer, List<InventoryVector> inventory)
Parameters
Type Name Description
INetworkPeer peer

Peer from which the message was received.
List<InventoryVector> inventory

List of inventory vectors to process.
Returns
Type Description
Task
Remarks

The "inv" message is sent unsolicited by legacy nodes to announce new blocks. It may contain non-block hashes which should be ignored. We then need to obtain the headers of the announced blocks so that the header processing can be performed as normal. We do not request the block directly as the peer may be attempting a DoS attack by spamming fake inv messages.
